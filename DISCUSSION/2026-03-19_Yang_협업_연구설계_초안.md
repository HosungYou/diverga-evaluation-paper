# Joint Research Design: You & Yang

## Designing Authentic AI Scaffolding for Research Methodology Learning: An LXD-Informed Design Case of a Multi-Agent Checkpoint System

**Hosung You** (Pennsylvania State University) & **Mohan Yang** (Texas A&M University)

**Status:** Draft for discussion — 2026-03-19

---

## 1. Research Problem

AI-powered research tools (e.g., Elicit, Consensus, ChatGPT) are rapidly being adopted by graduate students for research design tasks. However, two critical concerns remain unaddressed:

1. **Mode Collapse** — Generic AI tools converge on predictable, modal recommendations (e.g., TAM for technology adoption, thematic analysis for qualitative research), homogenizing methodological choices (arXiv:2510.01171).
2. **Automation Bias** — Studies report that 78% of agents accept AI output without independent review, and AI tool usage correlates negatively with critical thinking (Springer Nature, 2025; Microsoft Research, 2025).

The central question is: **Can AI research tools be designed to scaffold — rather than replace — researchers' methodological reasoning?**

This question is urgent because AI tools are being adopted faster than design frameworks can guide their responsible integration into research training.

## 3. Theoretical Framework

### 3.1 Integrated Framework

```
Layer 1: AUTHENTIC LEARNING (Herrington, Reeves & Oliver)
  → What makes AI-mediated research learning effective?
  → 9 elements as analytical and design framework

Layer 2: VERBALIZED SAMPLING + MODE COLLAPSE THEORY (arXiv:2510.01171)
  → Why do AI tools homogenize recommendations?
  → How does VS-based diversity generation counter this?

Layer 3: MCP-ORCHESTRATED MULTI-AGENT ARCHITECTURE
  → How does the technical architecture enable/constrain scaffolding?
  → Dynamic learner state (Yang, 2025) meets Human Checkpoints (You)
```

### 3.2 Authentic Learning as Design & Analysis Framework

Herrington, Reeves & Oliver's 9 elements of authentic learning provide both:
- **Design lens** — criteria for designing the Diverga checkpoint experience
- **Analytical lens** — dimensions along which to evaluate the scaffolding effect

| Authentic Learning Element | Diverga Design Feature | Design Rationale |
|---|---|---|
| Authentic context | Real research design scenarios | Ecological validity; not toy tasks |
| Authentic activity | Full RQ → theory → method pipeline | Matches actual researcher workflow |
| Multiple perspectives | VS multi-alternative generation + T-Score | Counters mode collapse; surfaces non-modal options |
| Collaboration | Structured AI-researcher dialogue at checkpoints | AI as epistemic partner, not oracle |
| Reflection | REQUIRED Human Checkpoints (forced pause) | Cognitive forcing function (Bucinca et al., 2021) |
| Articulation | Decision Audit Trail (YAML log) | Externalizes reasoning; supports metacognition |
| Coaching & scaffolding | Agent-specific contextual guidance + T-Score explanation | Calibrated support; fades as expertise grows |
| Authentic assessment | Research design artifact as product | Performance-based, not test-based |
| Expert performance modeling | Domain-specialized agents (44 across 9 categories) | Models expert reasoning in each domain |

### 3.3 Core Conjecture

> When an AI research tool implements authentic learning principles through multi-agent architecture with structured human checkpoints and diversity-generating mechanisms (VS), it scaffolds — rather than replaces — researchers' methodological reasoning. This scaffolding effect is mediated by three mechanisms: *forced reflection* (checkpoints), *perspective expansion* (VS alternatives), and *reasoning externalization* (decision audit trail).

## 5. Research Design

### 5.1 Two-Phase Design: Design Case + Focused Evaluation

This study adopts a two-phase design that combines:
- **Phase 1: LXD-Informed Design Case** — documenting the design rationale of Diverga's checkpoint system through the authentic learning framework
- **Phase 2: Mixed-Methods Evaluation** — empirically examining how the design performs with actual users

This approach follows Yang's established methodology (cf. Glaser et al., 2022; Yang et al., 2023) while incorporating the evaluation rigor needed to test the core conjecture.

### 5.2 Phase 1: Design Case (Weeks 1-4)

**Methodology:** Design Case (Boling, 2010; Smith, 2010) informed by Learning Experience Design (LXD)

**Purpose:** Systematically document how authentic learning principles were operationalized in Diverga's checkpoint architecture, including design decisions, alternatives considered, and rationale.

**Activities:**

| Activity | Method | Output |
|---|---|---|
| Authentic learning audit | Map each of 9 AL elements to Diverga features | AL-Diverga Alignment Matrix |
| Design decision documentation | LXD-structured retrospective of checkpoint design | Design Decision Log |
| MCP architecture analysis | Document how MCP enables/constrains each AL element | Architecture-Pedagogy Map |
| Expert review | 2-3 instructional design experts review the mapping | Expert Feedback Report |

**Yang's primary role in Phase 1:** Lead the authentic learning analysis and LXD framing; provide independent assessment of design decisions from an instructional design perspective.

**You's primary role in Phase 1:** Provide design rationale documentation; lead the MCP architecture analysis; ensure technical accuracy.

### 5.3 Phase 2: Focused Evaluation (Weeks 5-12)

**Methodology:** Mixed Methods — modeled on Yang et al. (2025, BJET ChatGPT study)

#### Participants

| Item | Detail |
|---|---|
| Population | Social science doctoral students |
| N | 12-16 (sufficient for mixed methods; cf. Creswell, 2018) |
| Sites | Penn State + Texas A&M (multi-site) |
| Inclusion | (1) Completed 1+ research methods course, (2) AI tool experience, (3) Basic CLI familiarity |
| Exclusion | (1) Prior Diverga users, (2) CS/Engineering majors |
| Recruitment | Guest workshops in research methods courses |

#### Task Design

**Pre-task (without Diverga):** Design a study on "The impact of AI tools on K-12 student academic achievement." Include theoretical framework, methods, and data collection plan. (30 min)

**Tool orientation:** Diverga features, checkpoint system, VS/T-Score explanation. (20 min)

**Post-task (with Diverga):** Design a study on "The impact of AI-based collaboration tools on team creativity in remote work." Include theoretical framework, methods, and data collection plan. Think-aloud protocol. (60 min)

**Re-design task:** Revisit pre-task scenario using Diverga. (40 min, think-aloud)

**Semi-structured interview:** (30 min)

#### Data Collection

| Data Source | Method | Analysis Purpose | Type |
|---|---|---|---|
| Checkpoint logs | Diverga decision-log.yaml | Selection patterns, deliberation time, T-Score choices | QUAN |
| Research design artifacts | Pre/post design documents | Framework diversity, design quality change | QUAN+QUAL |
| Think-aloud protocols | Screen + audio recording during checkpoint interactions | Cognitive processes triggered by checkpoints | QUAL |
| Semi-structured interviews | Post-session individual interviews | Perceived scaffolding, trust, design tensions | QUAL |
| Behavioral data | Screen recording | T-Score inspection frequency, navigation patterns | QUAN |

#### Analysis

**Quantitative:**
- Framework selection diversity: Shannon Entropy Index (pre vs. post)
- T-Score selection distribution: descriptive statistics
- Checkpoint deliberation time: descriptive statistics, pattern analysis
- Wilcoxon signed-rank test for pre-post comparisons (small sample)

**Qualitative:**
- Think-aloud transcripts: Thematic Analysis (Braun & Clarke, 2006) using authentic learning elements as sensitizing concepts
- Interview transcripts: Deductive coding with AL framework + inductive open coding
- Cross-case analysis across Penn State and Texas A&M participants

**Integration:**
- Joint display: Quantitative patterns (T-Score selection) alongside qualitative interpretation (why those choices)
- Convergence matrix: Where QUAN and QUAL findings agree/diverge

## 7. Expected Contributions

### Theoretical
1. **Authentic AI Scaffolding Framework** — First application of authentic learning theory to multi-agent AI research tools
2. **Checkpoint-as-Scaffold model** — Empirical evidence for how structured intervention points function as epistemic scaffolds (not just approval gates)
3. **MCP pedagogy** — Understanding how the Model Context Protocol architecture enables pedagogically grounded AI agent orchestration

### Practical
1. **Design principles** for AI research tool developers — evidence-based guidelines for checkpoint placement, VS integration, and scaffolding calibration
2. **Replicable evaluation methodology** for AI-mediated learning tools
3. **Multi-site validation** of findings across institutional contexts

### Methodological
1. **LXD-informed Design Case + Mixed Methods Evaluation** as a combined methodology for studying AI educational tools
2. **Decision Audit Trail as research data** — methodological precedent for using AI tool logs as behavioral data
